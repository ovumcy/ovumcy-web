# Ovumcy v0.7.1

Release date: 2026-03-22

## Highlights

- Added supported file-backed runtime secrets through `SECRET_KEY_FILE`, including operator docs, compose examples, and regression coverage.
- Added first-party French and German UI localization with parity across language switching, server-side date formatting, and onboarding segmented date labels.
- Moved public repository and documentation links to `github.com/ovumcy/ovumcy-web` and aligned tagged release images with the new GHCR namespace.

## Security and quality

- The runtime image and Go toolchain now use Go `1.25.8`, removing the vulnerable stdlib version previously flagged by Trivy.
- The transitive `flatted` dependency is updated to `3.4.2`.
- CI now treats external Codecov ingest failures on `push` as non-blocking so downstream smoke jobs still run.
- The release was validated with `go test ./...`, `staticcheck ./...`, `npm run lint`, `npm run build`, `npm run e2e -- e2e/navigation-language.spec.ts`, `gosec`, `docker build`, `docker compose config` for all supported stacks, and targeted runtime secret-source smoke checks.

## Upgrade notes

- No database migration is required for this release.
- Tagged images now publish under `ghcr.io/ovumcy/ovumcy-web:v0.7.1`.
- Existing deployments can upgrade in place and pin `OVUMCY_IMAGE=ghcr.io/ovumcy/ovumcy-web:v0.7.1` if they do not want to track future tags manually.
- `SECRET_KEY` remains supported and still takes precedence over `SECRET_KEY_FILE` when both are set.
