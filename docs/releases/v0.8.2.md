# Ovumcy v0.8.2

Release date: 2026-03-29

## Highlights

- Public patch release that supersedes `v0.8.1`.
- Includes the final CodeQL-driven hardening for the local OIDC browser-test runtime helper after the `v0.8.1` tag had already been published.

## Security and quality

- The local OIDC test helper no longer builds a server-rendered callback form with request-derived values. It now uses a constant HTML bridge page and a one-time JSON payload endpoint to preserve the hardened `form_post` browser flow without reflected callback markup.
- The patch release was revalidated with `npm run lint`, `git diff --check`, `docker compose config` for the supported release stacks, and all three OIDC browser lanes on Chromium, Firefox, and WebKit.
- The follow-up `main` workflow for commit `bcfe6f5` completed successfully across `CodeQL`, `CI`, `Security`, and `Docker Image`.

## Upgrade notes

- No new database migration is introduced beyond the schema changes already shipped in `v0.8.0`. If you are upgrading directly from `v0.7.2` or earlier, Ovumcy will still apply schema version `016_oidc_logout_states`.
- Tagged images publish under `ghcr.io/ovumcy/ovumcy-web:v0.8.2`.
- Existing deployments can upgrade in place and pin `OVUMCY_IMAGE=ghcr.io/ovumcy/ovumcy-web:v0.8.2` if they do not want to track future tags manually.
